Chapter 3: Creating and Verifying CDC Setup
Debugging CDC Setup Violations
Feedback
Potential Issues
If you do not fix this violation, the sequential elements which are reported are not checked 
during CDC/RDC analysis.
How to Debug
Sequential elements with clock pins tied to a constant are not checked during CDC/
RDC analysis. Not resolving such issues can cause valid crossings to be missed by 
the tool during CDC analysis. To analyze this violation, back trace the clock pin of the 
corresponding sequential logic.
Review the schematic to ensure that no constant is propagating to the clock pin of the cell. 
Review the 
set_case_analysis  constraints or any other constraint applied to the design.
Related Reason Code(s)
The tag reports the following reason codes only when below configuration is defined.
configure_cdc_setup_check -show_constant_source true
•
RTL_CONSTANT
• SCA_CONSTANT
• RTL_SCA_CONSTANT

Description
VC SpyGlass CDC reports this tag for top-level input ports, inout ports, and output 
ports which are fully constrained. A port is considered fully constrained if the following 
constraints are defined on it:
create_clock, set_input_delay, set_output_delay, set_case_analysis, creat
e_static, set_clock_attribute  or set_sync_attribute
Note:
By default, only constrained input ports are reported under this tag. To report 
output ports also along with the input ports, use the following command before 
the check_cdc  command:
configure_cdc_setup_check -check_port_setup all
For inout ports, if it is a data path, constraint the ports in both directions.

SETUP_CLOCK_GLITCH
Asynchronous source converges with different domain clock(s)
Severity
Error
When to Use
Use this tag to report asynchronous signals which isolates the clock and are in different 
domains with respect to the clock.
Prerequisites
• Specify the Elite  or Apex  license features
• Set the 
enable_cdc  application variable to true
• Read the design
• Specify clock signals using the create_clock  or create_generated_clock  SDC 
commands.
Description
VC SpyGlass CDC reports this tag to report asynchronous signals, which isolate the clock, 
but are not in the same domain as the clock signal. It reports one violation per clock cone 
per asynchronous isolation source signal.
A clock cone is a net that directly drives either the clock pin of a sequential element, top-
level port, or black-box pin.
