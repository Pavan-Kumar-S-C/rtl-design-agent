# Review workflow

**Keywords:** `review`, `code review`, `checklist`, `lint review`, `sign-off`

1. Use user-stated spec when provided; **ask only if doubtful** which spec applies.
2. Lint mindset: latch, CDC, reset, FSM completeness (load matching [topic standards](INDEX.md) or baseline guidelines on fallback).
3. Cross-check CSR and parameters vs RTL constants when CSR is clearly in scope.
4. Note cross-block impact when relevant — **ask** which blocks are in scope only if doubtful.
5. List open questions for user rather than guessing.
